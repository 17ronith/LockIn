### **Project Overview**
This project is a **multimodal video ranking system** designed to transform YouTube playlists into a focused, intent-aware video learning environment. It uses **textual and visual embeddings** to rank videos based on their relevance to a user's query or intent.

---

### **Project Structure**
1. **Key Files**:
   - video_ranker_multimodal.py: The main script for ranking videos using multimodal embeddings (text + visual).
   - video_ranker.py: A batch-processing version for ranking videos in large datasets.
   - finetune_data.csv: Dataset for fine-tuning the `SentenceTransformer` model.
   - videos.csv: Contains metadata for videos (e.g., titles, transcripts, thumbnail paths).
   - video_image_embeddings.npy: Precomputed visual embeddings for video thumbnails.
   - video_order.txt: Defines the order of videos to be synced with metadata.
   - my-finetuned-model: Directory containing the fine-tuned `SentenceTransformer` model.

2. **Directory Structure**:

### **Core Functionality**
1. **Multimodal Ranking**:
   - Combines **text embeddings** (from transcripts) and **visual embeddings** (from thumbnails) to rank videos.
   - Uses a weighted combination of text and visual similarity scores to compute a final ranking.

2. **Fine-Tuned Model**:
   - A `SentenceTransformer` model fine-tuned on finetune_data.csv for domain-specific queries.

3. **Ranking Logic**:
   - User intent is encoded using the text and visual models.
   - Cosine similarity is computed between the intent and video embeddings.
   - Videos are ranked based on a weighted combination of text and visual scores.

4. **Output**:
   - A ranked list of videos with metadata (title, scores, thumbnail path).

### **Extending the System for YouTube Playlists**
1. **User Input**:
   - A YouTube playlist URL.
   - A natural-language description of the user's intent.

2. **Playlist Parsing**:
   - Use the **YouTube Data API v3** to fetch metadata (titles, descriptions, thumbnails) for videos in the playlist.
   - Extract transcripts using third-party libraries like `youtube-transcript-api` (if captions are available).

3. **Handling Private Playlists**:
   - Request the user to make the playlist public.
   - Alternatively, implement OAuth authentication to access private playlists.

4. **Efficient Metadata Extraction**:
   - Use batch requests to fetch up to 50 videos per API call.
   - Cache playlist metadata locally to avoid redundant API calls.

### **Playing YouTube Videos in the Web Application**
1. **Use the YouTube IFrame Player API**:
   - Embed YouTube videos in your web application efficiently.
   - YouTube handles streaming, ensuring minimal lag and optimal quality.

2. **Implementation Example**:
   ```html
   <div id="player"></div>
   <script>
       var tag = document.createElement('script');
       tag.src = "https://www.youtube.com/iframe_api";
       var firstScriptTag = document.getElementsByTagName('script')[0];
       firstScriptTag.parentNode.insertBefore(tag, firstScriptTag);

       var player;
       function onYouTubeIframeAPIReady() {
           player = new YT.Player('player', {
               height: '390',
               width: '640',
               videoId: 'VIDEO_ID', // Replace with the YouTube video ID
               playerVars: {
                   'autoplay': 1,
                   'controls': 1,
                   'rel': 0,
                   'modestbranding': 1
               }
           });
       }
   </script>
   ```

3. **Advantages**:
   - Efficient streaming with no hosting overhead.
   - Customizable player (e.g., hide related videos, autoplay).
   - No copyright issues since videos are streamed directly from YouTube.

4. **Limitations**:
   - Requires an internet connection for playback.
   - Cannot modify or host YouTube videos directly due to copyright restrictions.

### **Focus and Learning Optimization**

   - Dynamically prioritize videos based on user intent.
   - Highlight the most relevant video at any moment.
