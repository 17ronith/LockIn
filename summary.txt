# LockIn Project - Complete Summary

## Project Overview

**LockIn** is a **multimodal YouTube playlist video ranking system** that intelligently reorders videos based on user intent by analyzing both textual content (titles/descriptions) and visual content (thumbnails) using machine learning embeddings. It then takes the video and it plays it within its own webapp, implementing distracion prevention mechanisms to help the user focus.

---

## What's Built

### 1. **Backend System** (Complete & Production-Ready)

#### Components:

**a) YouTube Data Integration (`playlist_parser.py`)**
- Fetches complete playlist metadata using YouTube Data API v3
- Extracts: video IDs, titles, descriptions, publish dates, thumbnails
- Implements JSON-based caching to avoid redundant API calls
- Supports batch requests for efficiency
- Currently handles playlists up to YouTube's pagination limits

**b) Multimodal Ranking Engine (`playlist_ranker.py` - FIXED)**
- **Text Embeddings**: Uses fine-tuned SentenceTransformer model (./my-finetuned-model) to convert video titles/descriptions into 384-dimensional vectors
- **Visual Embeddings**: Uses CLIP-ViT-B-32 model to convert video thumbnails into 512-dimensional vectors
- **Intelligent Thumbnail Handling**: 
  - Generates guaranteed-valid YouTube CDN URLs (https://i.ytimg.com/vi/{video_id}/{quality}.jpg)
  - Batch processes all images for consistency
  - Gracefully falls back to text-only scoring if thumbnails fail (no zero-vectors that corrupt rankings)
  - Tracks which videos have valid visual embeddings
- **Ranking Algorithm**: Combines scores with weighted formula: `final_score = (0.7 × text_score) + (0.3 × visual_score)`
- **Robustness**: Shape validation, comprehensive logging, graceful degradation

**c) FastAPI Backend (`api_backend.py`)**
- REST API running on http://localhost:8000
- **Endpoints**:
  - `POST /rank` - Submit playlist URL + user intent → returns ranked video list
  - `GET /health` - Health check with system status
  - `GET /docs` - Auto-generated Swagger documentation
- CORS enabled for frontend communication
- Returns full video metadata including:
  - Rankings with final/text/visual scores separately
  - Thumbnail URLs (3 quality levels)
  - YouTube links
  - `has_visual_embedding` flag indicating scoring method

**d) Models**
- **Fine-tuned SentenceTransformer** (./my-finetuned-model/): Custom trained on video description data for better context understanding
- **CLIP-ViT-B-32**: Pre-trained vision-language model for thumbnail analysis

#### How It Works:
1. User submits playlist URL + search intent (e.g., "learn machine learning")
2. Backend fetches all videos from playlist via YouTube API
3. Generates text embeddings for each video's title + description
4. Downloads all thumbnails using CDN URLs
5. Batch generates visual embeddings (ensures all same dimension)
6. Compares user intent embeddings against both text and visual embeddings
7. Scores each video with cosine similarity
8. Weights and combines scores (70% text, 30% visual)
9. Returns ranked list sorted by final score

### 3. **Problem Solved (Bug Fix)**

**Original Issue**: System only worked for one specific playlist, crashed on others

**Root Causes**:
1. Thumbnail URL inconsistency: Not all playlists return API's `thumbnail_url['default']` field
2. Mixed embeddings: Some videos had embeddings, some had zero-vectors → inhomogeneous array shapes
3. Fallback strategy: Zero-vectors corrupted rankings instead of gracefully degrading

**Solution Implemented**:
1. **CDN URL Generation**: Instead of relying on API responses, programmatically generate YouTube CDN URLs (guaranteed to exist for any valid video)
2. **Batch Processing**: Download all thumbnails first, encode them all at once → ensures consistent (N, 512) shaped array
3. **Graceful Degradation**: Track which videos have valid visual embeddings; use text-only scoring if visual fails (no zero-vectors)
4. **Shape Validation**: Validate embedding dimensions before tensor operations

**Result**: This error still persists, solution has not fixed the problem.

```
User Browser (http://localhost:3000)
         ↓
    [React Frontend - Vite]
         ↓ (POST /rank)
[FastAPI Backend on :8000]
         ↓
   [playlist_parser.py]
  YouTube API Key: YOUR_YOUTUBE_API_KEY
         ↓
   YouTube Data API v3
         ↓ (videos list)
   [playlist_ranker.py]
         ├─→ Text Embeddings (./my-finetuned-model)
         ├─→ Download Thumbnails (YouTube CDN)
         ├─→ Visual Embeddings (clip-ViT-B-32)
         └─→ Calculate Scores & Rank
         ↓
   Results (JSON)
         ↓ (response)
   [React Frontend]
         ↓
   Display Rankings to User
```
